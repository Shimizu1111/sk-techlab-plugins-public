# /plan - タスクプランニング

ユーザーのやりたいことを整理し、**必ずJSONファイルに保存する**スキル。
保存したプランは `/orchestrate` で1つずつ消化する。

**このスキルのゴールはファイル保存。保存せずに終わることは許されない。**

## 起動条件
ユーザーが `/plan` と入力したとき、または「計画して」「整理して」「やることリスト作って」等。

## 実行フロー

### Step 0: 既存プランの状態確認
1. `.claude/plans/` 内の `*.json` を確認
2. 既存プランがあれば、各プランの実行状態をサマリ表示:

```
## 既存プラン
| ファイル | ゴール | 状態 |
|----------|--------|------|
| 2026-05-14_auth.json | 認証機能追加 | 完了 (4/4) |
| 2026-05-14_ui.json | UI改善 | 進行中 (2/5) |
| 2026-05-14_refactor.json | リファクタ | 未着手 (0/3) |
```

**状態の判定:**
- `completed` フィールドがある → 「完了」
- タスクに `done`/`in_progress` があるが未完了あり → 「進行中 (done数/全数)」
- 全タスクが `pending` → 「未着手」

### Step 1: ヒアリング & コードベース確認
1. ユーザーの入力（雑でもOK）を受け取る
2. 必要ならコードベースを軽く確認（`Glob`, `Grep`, ファイル構成）
3. 以下を整理:
   - ゴール（何を達成したいか）
   - 現状（今どうなっているか）
   - 制約や優先度

### Step 2: タスク分解 & テーブル提示
各タスクをテーブル形式で提示:

```
## プラン: {ゴールの要約}

| # | タスク | タイプ | エージェント | 依存 | サイズ |
|---|--------|--------|-------------|------|--------|
| 1 | ... | research | gemini | - | S |
| 2 | ... | implementation | codex | 1 | M |
| 3 | ... | general | claude | - | S |
```

**振り分け基準:**
- `research`（調査・比較・ドキュメント読解）→ gemini
- `implementation`（複雑な実装・新機能・大規模変更）→ codex
- `general`（簡単な修正・設定・glue作業）→ claude

**サイズ:** `S`(数分) / `M`(10-30分) / `L`(30分+)

### Step 3: 確認 & 即保存
テーブル提示と同時に以下のように聞く:
```
修正があれば指摘してください。なければこのまま保存します。
```

- 修正指示があれば反映して再提示
- **「OK」「いい」「進めて」等の承認、または修正指示がない場合 → 即 Step 4**
- 「回答いただければ〜」のような冗長な確認は不要。シンプルに聞いてすぐ動く

### Step 4: ファイル保存（必須・スキップ不可）

**承認されたら確認なしで即保存する:**

1. `.claude/plans/` ディレクトリを作成（なければ）
2. JSONファイルを保存

ファイル名: `.claude/plans/{yyyy-MM-dd}_{slug}.json`

```json
{
  "goal": "ゴールの要約",
  "created": "2026-05-14T12:00:00",
  "tasks": [
    {
      "id": 1,
      "title": "タスク名",
      "description": "詳細（サブエージェントが単独で実行できるレベルの具体的な指示）",
      "type": "research | implementation | general",
      "agent": "gemini | codex | claude",
      "depends_on": [],
      "size": "S | M | L",
      "status": "pending",
      "context_summary": "このタスクに必要な最小限のコンテキスト"
    }
  ]
}
```

3. 保存完了を報告（**この文面を必ず出力する**）:
```
プランを保存しました: .claude/plans/{filename}.json
タスク数: {n}件（codex: {x} / gemini: {y} / claude: {z}）

次のステップ: `/orchestrate` でこのプランを実行できます。
```

## 重要ルール
- **ファイル保存は必須**。ユーザーが承認したら確認なしで即保存する
- タスクの `description` はサブエージェントがこれだけ読めば実行できるレベルで書く
- `context_summary` にはプロジェクト全体ではなくタスクに必要な情報だけ書く
- 依存関係は最小限にし、なるべく並行実行できるように分解する
- ユーザーが「これでいい」と言うまで何度でも修正する（ただし承認後は即保存）
