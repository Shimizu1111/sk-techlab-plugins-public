# /x-analytics - X投稿パフォーマンス分析

X MCPから投稿データを取得し、エンゲージメントを分析、前回比較と改善提案を行う。

## 引数

- `$ARGUMENTS` に対象アカウント名や期間を指定可能（省略可）
  - 例: `/x-analytics AI将軍`
  - 例: `/x-analytics 直近7日`

## 前提

- X MCPが `.mcp.json` に設定済みであること
- 分析結果は `SNS/{アカウント名}/分析/` に蓄積される
- 各アカウントの `SNS/{アカウント名}/X設定.md` にXユーザー名・分析対象開始日等の設定がある場合はそれに従う

## 実行手順

### Step 0: アカウント特定

`SNS/` 配下の**フォルダのみ**（.mdファイルは除外）を一覧表示し、番号で選択してもらう。
`$ARGUMENTS` にアカウント名が含まれていれば自動で特定してスキップ。

```
どのアカウントを分析する？

1. AI将軍
2. ai副業をハックするエンジニア
3. claude_code伴走支援
```

選択後、以下を読み込む:
1. **`アカウント設計.md`** — ターゲット・KPI・ポジショニングを把握
2. **`分析/`配下の直近ファイル**（あれば） — 前回データとの比較用

### Step 1: データ取得

X MCPを使って以下を取得する:

1. **プロフィール情報** (`get_user`)
   - フォロワー数、フォロー数、投稿数
2. **直近の投稿一覧** (`get_timeline`)
   - デフォルト: 直近100件を取得し、**前回分析日以降**のデータのみ使用
   - 前回分析がなければ直近7日分
3. **各投稿のメトリクス** (`get_metrics`)
   - impression_count, like_count, retweet_count, reply_count, quote_count, bookmark_count
   - ※ get_timelineで取れるpublic_metricsで十分な場合はget_metricsは省略してよい

### Step 2: データ整理・集計

取得データを以下の形式で整理する:

#### アカウントサマリー

```
フォロワー: {N} (+{前回比})
フォロー: {N}
投稿数: {N}
分析期間: {開始日} 〜 {終了日}
対象投稿数: {N}件
```

#### 投稿別パフォーマンス表

全投稿を一覧化する。以下の列を含む:

| 日時 | 種別 | 冒頭20文字 | Imp | Like | RT | Reply | Quote | BM | Eng率 |
|------|------|-----------|-----|------|----|-------|-------|----|-------|

- **種別**: 親ツイート / スレッド返信 / リプライ を判定
  - `conversation_id` が自分の `id` と一致 → 親ツイート
  - `in_reply_to_user_id` が自分 → スレッド返信
  - `in_reply_to_user_id` が他者 → リプライ
- **Eng率**: (like + RT + reply + quote + bookmark) / impression * 100

#### 集計指標

以下を算出する:

- **平均インプレッション**: 親ツイートのみ / 全体
- **平均エンゲージメント率**: 親ツイートのみ / 全体
- **RT率**: RT数 / 投稿数
- **プロフィール誘導率**: (推定) いいね+リプライしてくれた人のうち新規の割合
- **最もインプレッションが高い投稿 Top3**
- **最もエンゲージメント率が高い投稿 Top3**
- **投稿時間帯別パフォーマンス** (朝/昼/夕/夜)

### Step 3: 分析・診断

以下の観点で分析し、**現状の課題**を特定する:

#### ファネル分析
```
インプレッション → 詳細クリック → エンゲージメント → プロフィール訪問 → フォロー
```
どこで詰まっているかを特定する。

#### コンテンツ分析
- **どのテーマ/切り口が伸びているか**: 実体験 vs 海外事例 vs 意見系 etc.
- **どの構成が効いているか**: 単発 vs スレッド、問題提起型 vs How-to型 etc.
- **スレッドの減衰率**: 親ツイート → 2つ目 → 3つ目のインプレッション推移

#### アカウント設計とのギャップ
- KPIとの乖離（フォロワー目標 vs 現在値）
- コンテンツ4本柱の実際の投稿比率
- ターゲット層にリーチできているか

### Step 4: 改善提案

分析結果に基づいて、**優先度付きで3〜5個の具体的な改善アクション**を提示する。

提案フォーマット:
```
## 改善提案

### 1. [最優先] {アクション名}
- 現状: {データに基づく問題}
- 施策: {具体的にやること}
- 期待効果: {改善が見込める指標}

### 2. [高] {アクション名}
...
```

提案は以下を心がける:
- データに基づいた根拠を示す
- 「何をやめるか」も含める（やることを増やすだけにしない）
- アカウント設計のKPIに紐づける

### Step 5: レポート保存

分析結果を以下のファイルに保存する:

- **保存先**: `SNS/{アカウント名}/分析/{YYYYMMDD}_x分析.md`
- 同じ日付のファイルがあれば上書きしてよいか確認する

#### ファイルフォーマット

```markdown
---
date: {YYYY-MM-DD}
platform: X
account: {アカウント名}
period: {開始日} 〜 {終了日}
followers: {N}
total_posts: {分析対象投稿数}
---

# X分析レポート {YYYY-MM-DD}

## アカウントサマリー
{Step 2のサマリー}

## 投稿パフォーマンス
{Step 2の表}

## 集計指標
{Step 2の集計}

## 分析・診断
{Step 3の内容}

## 改善提案
{Step 4の内容}

## 前回比較
{前回データがあれば差分を記載}
```

### Step 6: ネクストアクション確認

レポート保存後、ユーザーに聞く:

**「改善提案のうち、今すぐやれるものはどれ？ 投稿ルール（x-draft）に反映する？」**

ユーザーが投稿ルールへの反映を希望した場合は、x-draftスキルの`references/posting-rules.md`に改善ポイントを追記する。

## 使用例

```
/x-analytics
→ アカウント選択 → データ取得 → 分析 → 改善提案 → レポート保存

/x-analytics AI将軍
→ AI将軍アカウントを直接分析

/x-analytics AI将軍 直近30日
→ 期間指定で分析
```
