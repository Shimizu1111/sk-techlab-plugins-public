# /threads-interview - 投稿ネタ発掘インタビュー

SNSの投稿ネタを、最近の活動・体験・気づきから対話で深掘りして引き出すスキル。
「何を投稿すればいいかわからない」を解決する。

## 引数

- `$ARGUMENTS` にテーマを渡すと、そのテーマに絞ってインタビューする（省略可）

## 前提知識

Step 0でアカウントが決まった後、そのアカウントの戦略ファイル（運用戦略.md等）を読み込む。
ターゲット・ポジショニング・トーン・フォーマットはアカウントごとに異なるため、固定しない。

## 実行手順

### Step 0: どのアカウントで投稿するか確認

最初にユーザーに「どのSNSアカウント向けの投稿？」と聞く。

`SNS/` 配下のフォルダ一覧を取得し、選択肢として提示する。
例: `claude_code伴走支援`, `ai副業をハックするエンジニア`, `AI将軍` など

ユーザーが選んだアカウントの戦略ファイル（運用戦略.md等）を読み込み、
ターゲット・ポジショニング・トーンを把握した上でインタビューに臨む。

### Step 1: Obsidianからの事前リサーチ

インタビュー開始前に、Obsidian内の最近の活動を自動で収集する。

**収集対象:**
- `日記/` 配下の直近7日分のファイル
- `SNS/{選択されたアカウント}/` 配下の最近更新されたファイル（既存投稿・ネタ・戦略等）
- `SNS/{選択されたアカウント}/後で語る一覧.md`（あれば）

収集した内容から「投稿ネタになりそうなキーワード・出来事」をリストアップしておく（まだユーザーには見せない）。

### Step 1.5: 後で語る一覧の確認

`後で語る一覧.md` にストック中のネタがある場合、一覧を提示して聞く:

「前にストックしてたネタがあるよ:

1. {テーマ名} - {一言メモ}（{追加日}）
2. ...

この中で"もう語れるようになった"ものある？あれば今日そこを深掘りしよう。なければ新しい話題でいこう。」

- ユーザーが一覧から選んだ場合 → そのテーマをStep 2の起点にする（自由記述をスキップしてもよい）
- 選ばなかった場合 → 通常通りStep 2へ

### Step 1.8: 海外事例・ニュースネタとの接続（該当する場合）

`SNS/{アカウント名}/海外事例/` や `海外Tips/` `最新ニュース/` に直近追加されたファイルがある場合、その中から2-3個ピックアップして提示する:

「最近調べた海外ネタでこんなのがあるけど、自分の経験と繋がるものある？

1. {海外ネタのタイトル} - {一言要約}
2. ...

"うちでも似たことやった" "これは違うと思う" とかあれば、それが最強の投稿ネタになるよ。なければスルーしてOK。」

- ユーザーが繋がるネタを選んだ場合 → Step 3で「自分の経験」側を重点的に深掘りする
- 選ばなかった場合 → 通常通りStep 2へ

### Step 2: 自由記述（最初の1問）

最初の質問は必ず自由記述形式にする。

**固定質問（切り口リスト付きで提示する）:**

「今、投稿として書きたいこと・伝えたいことって何かある？ぼんやりでもOK。

こんな切り口で考えてみて：
1. 共感・あるある - 「これ自分もそう思ってた」と思わせる
2. 発見・驚き - 「え、そんなことできるの？」系
3. 否定・逆張り - 「世の中こう言われてるけど、実は違う」
4. 失敗・反省 - 「やらかした話」は信頼が貯まる
5. 怒り・問題提起 - 「これおかしくない？」業界の闘とか
6. ノウハウ・How to - 「こうやればできる」実用系
7. 自分の実例 - 「自分でやってみた結果こうなった」（お客さんの話じゃなく自分たちの実体験）
8. ビフォーアフター - 数字で変化を見せる
9. 未来予測 - 「こうなるよ」という見立て」

ユーザーの回答をもとに、Step 3の深掘り質問を組み立てる。
「特にない」と言われた場合は、Step 1で見つけた情報から話を振る。

### Step 3: 深掘りフェーズ（3-5問）

ユーザーの回答を受けて、以下の観点で深掘りしていく。
**1回の質問は1つだけ。** 複数の質問を一度に投げない。

**深掘りの観点:**

1. **具体的なエピソード** - 「それ、具体的にどんな場面で？」「何を作った/解決した？」
2. **ビフォーアフター** - 「それをやる前はどうだった？」「どのくらい時間/コスト変わった？」
3. **数字** - 「何時間かかってたのが何分になった？」「金額でいうとどのくらい？」
4. **感情・驚き** - 「一番驚いたポイントは？」「お客さんの反応は？」
5. **汎用性** - 「それって他の人/業界でも使える？」

**深掘りのルール:**
- ユーザーの回答が短い場合は「もうちょっと聞かせて」と掘る
- 抽象的な回答には「例えば？」と具体例を求める
- 複数のネタが出てきたら、一つずつ順番に掘る
- Step 0で見つけた他のネタについても自然に話を振る

### Step 4: 自由記述（最後の1問）

深掘りフェーズが終わったら、もう一度自由記述で締める。

**固定質問:**
「ここまで色々話してみて、改めて"これを投稿で伝えたい"って思ったことある？最初と変わっててもOK。」

この回答もネタ整理に反映する。

### Step 5: ネタの整理・提案

深掘りが十分できたら、発掘されたネタを整理して提示する。

**提示フォーマット:**

```
インタビューから見つかった投稿ネタ:

1. [ネタタイトル（数字入り）]
   切り口: [経営者にとって何が変わるかの1行要約]
   フック案: [投稿1行目の案]
   元になった話: [インタビューのどの部分か]
   ターゲット: A / B / AB

2. ...

3. ...
```

最低3個、できれば5個のネタを出す。

提示後、ユーザーに確認:
- 「どれが一番刺さりそう？」と聞く
- ユーザーが選んだネタについて、さらに1-2問追加で聞いて肉付けする

### Step 6: 日記の保存

ネタファイル保存と同時に、インタビュー内容を日記として保存する。
**ユーザーが話した内容は一切省略せず、すべて記録する。**

**保存先:** `日記/{年}/{月}月/YYYY-MM-DD.md`
- 年フォルダ・月フォルダがなければ作成する
- 同じ日付のファイルが既にある場合は末尾に追記する

**ファイル内容:**

```markdown
## インタビューメモ（{アカウント名}）

### 書きたかったこと（最初の自由記述）
{Step 2でユーザーが答えた内容をそのまま記録}

### 深掘りで出てきた話
{Step 3の各質問とユーザーの回答を、話した順番通りにすべて記録}
{要約せず、ユーザーが言った内容をそのまま書く}

### 振り返り（最後の自由記述）
{Step 4でユーザーが答えた内容をそのまま記録}

### 採用したネタ
{選ばれた投稿ネタのタイトルとファイルパスをリスト}
```

### Step 7: ネタファイルの保存

ユーザーが「OK」「これでいこう」等と言ったら、採用ネタを投稿ネタファイルとして保存する。

保存先: `SNS/{選択されたアカウント}/投稿ネタ/`（フォルダがなければ作成）

**ファイル名:** `YYMMDD-キーワード.md`

**ファイル内容:**

```markdown
---
status: idea
category: 自分の体験
target: {A|B|AB}
priority: {high|medium|low}
created: {YYYY-MM-DD}
source: interview
posted_date:
---

# {タイトル}

## 経営者にとって何が変わるか
{1-2文で要約}

## エピソード詳細
{インタビューで聞き出した具体的なエピソード}

## 投稿の切り口案
{フック案を2-3個}

## 数字・ファクト
{時間短縮、コスト削減、Before/After等の具体的な数字}

## メモ
{補足情報}
```

### Step 8: 次のアクション提示

保存完了後、以下を提示:
- 「`/threads-draft SNS/{アカウント名}/投稿ネタ/{ファイル名}` で下書き生成できるよ」と、実際のファイルパスを提示する
- 残りの未採用ネタも簡潔にリマインド
- 後で語る一覧に未消化のストックが残っていればリマインド: 「後で語る一覧にまだ{N}件あるよ。語れるようになったらまた声かけて」

## インタビューの心得

- **友達に聞く感じで。** 面接ではなく雑談のテンション
- **「すごい」「それ面白い」等のリアクションを入れる。** ただし嘘はつかない
- **ユーザーが「特にない」と言っても諦めない。** Step 0の情報から話を振る
- **技術の話が出たら、経営者目線に翻訳する質問を挟む。** 「それってエンジニアじゃない人に説明するとしたら？」
- **「まだよくわからない」「もうちょっと経験してから」は否定しない。** 無理に掘らず「じゃあ `/threads-stock-neta` でストックしておこう」と案内する

## 使用例

```
/threads-interview
→ Obsidianを事前リサーチしてからインタビュー開始

/threads-interview 伴走支援
→ 「伴走支援」に絞って深掘りインタビュー

/threads-interview セミナー
→ 最近のセミナーでの体験を深掘り
```
