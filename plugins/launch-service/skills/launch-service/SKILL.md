# /launch-service - SaaS/Webサービス立ち上げチェックリスト

対話形式でサービス立ち上げに必要な法務・ビジネス・技術面のチェックを段階的に進め、各ステップの完了を確認しながらローンチ可否を判定するスキル。

## 起動時の動作

1. ユーザーにサービスの概要を聞く:
   - サービス名
   - サービス概要（何を提供するか）
   - ターゲット市場（日本/グローバル/特定地域）
   - 課金モデル（サブスク/従量/買い切り/フリーミアム/無料）
   - 想定ユーザー（個人/法人/両方）
   - 個人情報の取り扱い有無

2. 回答に基づき、該当するステップのみを抽出してカスタムチェックリストを生成する

## チェックリスト全体像

以下の8フェーズで構成。各フェーズ内のステップを1つずつ対話で進める。

---

### Phase 1: 法的リスク調査

- [ ] **特許・知的財産の調査**
  - 類似サービスの特許出願状況を調査（J-PlatPat, Google Patents）
  - 商標の被りを確認（同名・類似名のサービスが存在しないか）
  - ドメイン名の確保状況を確認
  - OSSライセンスの確認（使用するライブラリのライセンス互換性）

- [ ] **業法・規制の確認**
  - 該当する業法がないか確認（金融・医療・教育・通信など）
  - 許認可・届出が必要か確認
  - 資金決済法への該当有無（ポイント・前払式支払手段など）
  - 電気通信事業の届出要否

### Phase 2: 利用規約・ポリシー策定

- [ ] **利用規約（Terms of Service）**
  - サービス内容の定義
  - アカウント作成条件・年齢制限
  - 禁止事項
  - 知的財産権の帰属（ユーザー生成コンテンツ含む）
  - 免責事項・損害賠償の制限
  - 利用規約の変更手続き
  - 準拠法・管轄裁判所
  - 反社会的勢力の排除条項

- [ ] **プライバシーポリシー**
  - 収集する個人情報の種類と利用目的
  - 第三者提供の有無と条件
  - Cookie・トラッキングの使用
  - データ保存期間と保存場所
  - ユーザーの権利（開示・訂正・削除請求）
  - 個人情報保護委員会への届出要否
  - グローバル展開時: GDPR/CCPA対応

- [ ] **特定商取引法に基づく表記**（課金ありの場合）
  - 事業者名・代表者名
  - 所在地
  - 連絡先（電話番号・メールアドレス）
  - 販売価格（税込表示）
  - 支払方法
  - 商品の引渡し時期（サービス提供開始時期）
  - 返品・キャンセルに関する事項

### Phase 3: 決済・課金設計（課金ありの場合）

- [ ] **決済手段の選定**
  - 決済プロバイダの選定（Stripe / PAY.JP / Square など）
  - 対応する決済手段（クレカ / コンビニ / 銀行振込など）
  - PCI DSS準拠の確認（トークン化等で非保持化）

- [ ] **サブスクリプション設計**（サブスクの場合）
  - 無料トライアル期間の設計
  - プラン変更（アップグレード/ダウングレード）のルール
  - 日割り計算の有無

- [ ] **解約・返金ポリシー**
  - 解約手続きの方法（ユーザーが自分でできるか）
  - 解約時のデータ保持期間
  - 返金条件と手続き
  - 決済停止のタイミング（即時/期間末）
  - 未払い時の対応フロー

- [ ] **インボイス制度対応**（日本向け法人課金の場合）
  - 適格請求書発行事業者の登録
  - 請求書のフォーマット要件

### Phase 4: セキュリティ・データ保護

- [ ] **セキュリティ基本対策**
  - HTTPS必須化
  - 認証・認可の設計（OAuth / パスワードポリシー）
  - OWASP Top 10への対策
  - 脆弱性診断の実施（またはスケジュール）
  - インシデント対応手順の策定

- [ ] **データ保護**
  - バックアップ戦略
  - データ暗号化（保存時・通信時）
  - ログの保存と監視
  - データ削除手順（退会時）

### Phase 5: 運用体制

- [ ] **カスタマーサポート体制**
  - 問い合わせ窓口の設置（メール / フォーム / チャット）
  - 対応時間の明示
  - FAQ・ヘルプページの整備

- [ ] **障害対応体制**
  - 監視体制の構築
  - SLA（サービスレベル合意）の定義
  - ステータスページの用意
  - 障害時の通知手段

- [ ] **運用ドキュメント**
  - 運用手順書
  - エスカレーションフロー
  - 定期メンテナンスのスケジュール

### Phase 6: ビジネス・会計

- [ ] **事業形態の確認**
  - 個人事業 / 法人の選択
  - 開業届・法人設立の手続き
  - 事業用口座の準備

- [ ] **会計・税務**
  - 売上計上基準の決定
  - 消費税の扱い（課税/免税/軽減税率）
  - 海外売上時の源泉税・VAT対応
  - 会計ソフトとの連携

- [ ] **保険・リスクヘッジ**
  - サイバー保険の検討
  - 賠償責任保険の検討

### Phase 7: マーケティング・ローンチ準備

- [ ] **ローンチ前準備**
  - LP（ランディングページ）の作成
  - 事前登録・ウェイトリストの仕組み
  - ベータテストの実施

- [ ] **法的表示の配置確認**
  - 利用規約へのリンク（登録フロー内）
  - プライバシーポリシーへのリンク（フッター等）
  - 特商法表記ページ（課金ありの場合）
  - Cookie同意バナー（必要な場合）
  - 同意チェックボックスの配置

### Phase 8: ローンチ判定

- [ ] **最終チェック**
  - 全フェーズの必須項目が完了しているか
  - 法的文書のレビュー（可能なら弁護士確認）
  - テスト決済の動作確認（課金ありの場合）
  - 本番環境でのセキュリティ確認
  - ローンチ後の改善ロードマップの策定

---

## 対話の進め方

1. 各フェーズの開始時に、そのフェーズの概要と含まれるステップを提示
2. 各ステップで以下を行う:
   - 何をすべきかの説明
   - 必要に応じて具体的なテンプレートや参考情報を提供
   - ユーザーの状況を聞き、対応済み/未対応/該当なしを確認
3. 未対応の場合はその場で対応を支援（文書のドラフト作成など）
4. 全ステップ完了後、ローンチ可否のサマリーを出力

## 出力フォーマット

進捗は以下の形式で表示:

```
=== サービス立ち上げチェックリスト: [サービス名] ===

Phase 1: 法的リスク調査        [2/2 完了]
Phase 2: 利用規約・ポリシー策定  [1/3 完了]
Phase 3: 決済・課金設計         [0/4 未着手]
...

現在: Phase 2 > プライバシーポリシー
```

## 注意事項

- このスキルは法的助言を提供するものではない。最終的な法的判断は弁護士等の専門家に確認を推奨する
- 業種・規模・地域によって必要な対応は異なる。該当しないステップはスキップ可能
- 課金なしのサービスの場合、Phase 3 と特商法表記はスキップされる
