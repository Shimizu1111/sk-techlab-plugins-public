# /threads-news-research - 最新ニュースリサーチ

直近1週間のClaude Code・AI関連の最新ニュースを収集し、**指定アカウントのターゲットに刺さる形**で蓄積する。threads-draft / x-draft / fb-draft に渡せる投稿ネタとして保存する。

**鮮度が命。古い情報に価値はない。**

## 引数

- `$ARGUMENTS` にアカウント名やテーマを渡せる（省略可）

## 実行手順

### Step 0: アカウント選択 & ターゲットセグメント選択

#### 0-1. アカウント選択

`SNS/` 配下のフォルダ一覧を取得し、ユーザーに選んでもらう。
`$ARGUMENTS` にアカウント名が含まれていれば自動で特定してスキップ。

#### 0-2. ターゲットセグメント選択

以下の3段階からユーザーに選んでもらう。`$ARGUMENTS` に含まれていれば自動特定。

| セグメント | 説明 | 関心事 |
|-----------|------|--------|
| **検討段階** | Claude Codeをこれから使い始めようとしている | 導入すべきか、ROI、リスク、他社の成功/失敗、何ができるか |
| **初期段階** | 使い始めたけどいまいち使いきれていない | なぜうまくいかない、すぐできる改善、正しい使い方、期待値調整 |
| **活用段階** | 割と使いこなしている | さらに効果を出す方法、チーム展開、自動化拡大、高度な活用 |

**このセグメント選択がリサーチ全体の方向性を決める。** Step 3のクエリ設計、Step 4のフィルタリングすべてに影響する。

### Step 1: アカウント文脈の読み込み

選択されたアカウントの以下を読み込む:

1. **`アカウント設計.md`** -- ターゲット・ポジショニング・トーン・コンテンツ4本柱
2. **`自社情報.md`**（あれば） -- 自分の事業との接点を見つけるため

**読み込んだ情報から以下を確認し、リサーチの軸にする:**

- **ターゲットは誰か**（例: 非ITの経営者・意思決定者）
- **ターゲットの関心事は何か**（例: 売上・コスト・人材・時間）
- **ターゲットに刺さる言葉は何か**（例: 経営者の言葉、具体的な数字と業務名）
- **ターゲットに刺さらない言葉は何か**（例: 技術用語、エンジニア向けの話）

### Step 2: テーマ確認

ユーザーが指定したテーマがあればそれを使う。なければ「Claude Code・AI全般の直近1週間のニュース」で進める。
迷ったら一言だけ確認する（例: 「新機能系・業界動向系・活用事例系のどれ寄せます？」）。長々と質問しない。

### Step 3: ターゲット視点でクエリを設計し検索

**重要: Step 1で読み込んだターゲット像に基づいてクエリを設計する。**
技術的なリリースノートだけでなく、ターゲットの関心事（コスト・売上・人材・導入判断）に関わるニュースも探す。

WebSearch を **複数クエリ並列** で投げる。**日本語ソースも英語ソースも両方対象**。

**必ず直近1週間に絞る**。古い情報は無価値。

クエリ例（英語 - 基本・全セグメント共通）:
- `Claude Code news 2026`
- `Claude Code update announcement`
- `Anthropic Claude announcement this week`
- `"Claude Code" release`
- `Anthropic blog 2026`

#### セグメント別の追加クエリ

**検討段階向け（導入判断に関わるニュース）:**
- `Claude Code pricing change 2026`
- `Anthropic enterprise plan announcement`
- `Claude Code new business adoption`
- `AI coding tool market growth 2026`

**初期段階向け（使い始めた人に影響するニュース）:**
- `Claude Code usability improvement`
- `Claude Code beginner feature update`
- `Anthropic Claude Code onboarding`

**活用段階向け（活用を広げるニュース）:**
- `Claude Code advanced feature announcement`
- `Claude Code team collaboration update`
- `Claude Code enterprise feature`
- `Anthropic API new capability`

クエリ例（日本語）:
- `Claude Code 最新 ニュース`
- `Claude Code アップデート`
- `Anthropic 発表`
- `Claude Code 新機能`
- `Claude Code 導入 企業`

優先ソース:
- Anthropic公式ブログ・リリースノート・X公式アカウント
- TechCrunch / The Verge / Ars Technica / VentureBeat
- 日本のテック系メディア（ITmedia, TechCrunch Japan, GIGAZINE, Publickey等）
- HackerNews（速報性が高い）
- Reddit (r/ClaudeAI, r/Anthropic)
- X / Twitter（AI関連の速報）
- GitHub Releases / Changelog

### Step 4: ニュースフィルタリング

検索結果を **ニュースルール** に基づいて選別する。タイトルだけで判断せず、WebFetch で中身を確認してから判断する。

## ニュースルール（調査の独自ルール）

### 0. ターゲット立場テスト & セグメントマッチ（全ルールに優先・最初に判定）

ニュースを見つけたら、他の判定の前に2つ問う:

**Q1: 「このニュースは、ターゲットの経営判断・行動に影響するか？」**
**Q2: 「Step 0-2で選んだセグメント（検討/初期/活用）の人に刺さるか？」**

#### セグメント別の保存/スキップ基準

| セグメント | 刺さるニュース | 刺さらないニュース |
|-----------|----------|----------|
| **検討段階** | 価格改定、大型導入事例、市場成長データ、「始めるなら今」系 | 既存ユーザー向けの機能改善、高度な新機能 |
| **初期段階** | 使いやすさ改善、初心者向け新機能、サポート充実 | 導入を促すデータ（もう使っている）、上級者向け新機能 |
| **活用段階** | 高度な新機能、エンタープライズ機能、API拡張、チーム機能 | 基本的な使い方の話、導入を促すデータ |

#### 具体例

**検討段階の人に対して:**
- 「Claude Codeの料金が○○に変更」→ **保存**（導入コストの判断材料）
- 「○○社がClaude Code全社導入で外注費50%削減」→ **保存**（導入の後押し）
- 「サブエージェントのコンテキスト管理が改善」→ **スキップ**（まだ使っていない）

**初期段階の人に対して:**
- 「Claude Codeの初期設定が簡素化された」→ **保存**（使いやすくなる情報）
- 「Anthropicの企業価値が○○に」→ **スキップ**（もう使っている、導入判断は済み）

**活用段階の人に対して:**
- 「動的ワークフロー機能が追加」→ **保存**（さらに活用を広げられる）
- 「Claude Codeとは何か」的な紹介記事→ **スキップ**（既に知っている）

**「翻訳すれば刺さる」は言い訳にしない。** セグメントに合わないニュースは、言い換えても行動が変わらない。

### 1. 鮮度フィルター（最重要）

**直近1週間以内に公開・発表されたものだけを対象とする。**

- 公開日が確認できないものは、内容の新しさから推測し、古そうならスキップ
- 「○月○日に発表された」等の言及があるものを優先
- 同じニュースの二次報道が複数ある場合は、**一次ソース（公式発表）を優先**

### 2. ニュース性フィルター

「ニュース」として価値があるものだけを残す。

| 対象 | 例 |
|------|-----|
| 新機能・アップデート発表 | 「Claude Codeに○○機能が追加」 |
| 公式の方針変更・価格改定 | 「Anthropicが○○を発表」 |
| 大型の提携・導入事例 | 「○○社がClaude Codeを全社導入」 |
| 業界を動かすトレンド | 「AIコーディング市場が○○に」 |
| 規制・法律の動き | 「EU/日本政府がAI○○を決定」 |

| 対象外 | 理由 |
|--------|------|
| 個人のTips・ハウツー記事 | ニュースではない（/threads-global-researchの領域） |
| ポエム・オピニオン記事 | 事実ベースではない |
| 3ヶ月以上前のまとめ記事 | 古い |
| リリースノートの軽微な修正 | ニュース性が低い |

### 3. ターゲットフィルター

ニュースを見つけたら問う: **「これを聞いたターゲットが、翌日の行動を変えるか？」**

- 「AI導入を検討し始める」「予算を確保する」「チームに指示を出す」→ 保存対象
- 「へー」で終わる → **スキップ**（「翻訳すれば刺さる」は言い訳にしない）
- 技術者しか分からない → **Rule 0でスキップ済みのはず。すり抜けていたらここでスキップ**

**注意:** 「経営者の言葉に言い換えれば伝わる」は保存理由にならない。本質的にターゲットの関心事（売上・コスト・人材・時間・リスク）に影響するニュースだけを保存する。

### 4. 翻訳の方向性

**翻訳はフィルタリングの後に行う。** Rule 0・3を通過したニュースだけを翻訳する。翻訳で「刺さるように見せかける」のではなく、本質的にターゲットに関係あるニュースを、ターゲットの言葉で伝える。

```
x Rule 0でスキップすべきニュース（翻訳しても意味がない）:
「Claude Code 1.0.xx でサブエージェントのコンテキスト管理が改善」
→ 言い換えても経営者の行動は変わらない

o Rule 0を通過するニュース（翻訳する価値がある）:
「Claude Code の料金プランが改定。月額60%オフの新プラン登場」
→ 「AI社員の"給料"が月5万円から月2万円に。導入ハードルが一気に下がった」
```

翻訳のポイント:
- **そもそもターゲットの関心事に合致するニュースを選んでから翻訳する**
- **「だから社長のあなたに何が起きるか」まで書く**
- **数字があれば必ず含める**（コスト・時間・人数）

### 5. 具体性の水準

| 優先度 | 条件 | 例 |
|--------|------|-----|
| A | 数字+金額がある | 「月額○円値下げ」「処理速度○倍」 |
| B | 数字はあるが金額なし | 「○%高速化」「○社が導入」 |
| C | 具体的な機能名・社名がある | 「○○機能を追加」「○○社と提携」 |
| D | 抽象的 | 「AIが進化した」「便利になった」 |

**D（抽象的）のニュースは原則スキップ**。

### 6. 需要チェック

そのニュースが **ターゲットにとって「知りたい」情報かどうか** を判断する。

- **「これを知らないと損する」** → 最優先（例: 価格改定、重要な新機能）
- **「知っておくと得する」** → 優先（例: 業界トレンド、大型導入事例）
- **「知ってても知らなくても変わらない」** → スキップ

### 7. 速報性ボーナス

- **日本語メディアでまだ報じられていないニュース** → 優先度を1段階上げる
- **公式発表から24時間以内** → 「速報」タグを付ける
- 「第一報を届ける」価値がある情報を優先する

---

### Step 5: ニュースごとに保存

1ニュース = 1ファイル。3〜5件を目安。

保存先: `SNS/{アカウント名}/最新ニュース/YYYY-MM-DD_<短い英数タイトル>.md`
（ファイル名はOS互換性のため英数字+ハイフン推奨。日本語タイトルはフロントマター/本文に書く）

### Step 6: サマリー返答

最後に以下を返答する:
- 収集したニュースの一覧（タイトル + 1行要約 + 公開日）
- 特に **速報性が高い・今すぐ投稿すべき** 1〜2本のおすすめ理由
- 「この話題は○日以内に投稿しないと鮮度が落ちる」等のタイミングアドバイス
- **「/threads-draft や /x-draft で下書きにしますか？」** と次のアクションを促す

```markdown

  - https://...

# <ターゲットに刺さる日本語タイトル>

## 概要（ターゲット向け）
3〜5文で、**ターゲットの言葉で**書く。技術用語は使わない。
「何が起きたか・あなたにとって何が意味あるか」の順。

## ニュースの事実
- 発表元・発表日
- 何が発表/変更されたか（正確に記録）
- 定量データ（数字がなければ「言及なし」と明記）
- 一次ソースURL

## ターゲットへの翻訳
- 技術→業務インパクトへの言い換え
- 「だから社長のあなたに何が起きるか」
- 「今すぐ何かすべきか、様子見でいいか」の判断

## SNS投稿ネタ候補

### 投稿の核（1行で）
> このニュースから伝えたい1メッセージ

### フック候補（2〜3案）
> ターゲットのスクロールを止める冒頭文を2〜3案。
> アカウントのトーンに合わせて書く。
> 技術用語は使わない。
> 速報性を活かす表現を入れる（「昨日発表された」「今週○○が」等）。

### 本文の骨子
- 導入（フック）→ ニュースの紹介 → 翻訳（あなたにとっての意味）→ 着地
- 着地はターゲットの関心事（売上・コスト・人材・時間）に

## ソース
- [タイトル](URL) - 公開日
- 関連: [タイトル](URL)

## 投稿タイミング
- 推奨投稿期限: YYYY-MM-DD（この日を過ぎると鮮度が落ちる）
- 理由: <なぜその期限か>
```

## ルール

- **鮮度が最重要**。直近1週間以内のニュースだけを対象にする。例外なし。
- **ターゲットの言葉で要約する**。技術用語はそのまま残さず、業務の言葉に翻訳する。ただしツール名（Claude Code等）はそのまま残す。
- **数字や固有名詞を捏造しない**。確認できないものは「未確認」「ソース上の言及なし」と書く。
- **一次ソースを優先**。二次報道・まとめ記事よりも公式発表を引用する。
- **引用は短く**。長文引用は避け、要約 + 短い引用 + ソースURLで構成する。
- **ネタとして弱い場合は無理に保存しない**。「今週は大きなニュースが無かった」と素直に伝える。
- **保存先フォルダが無ければ作成**してから書き込む。
- **ニュースルール1〜7をすべて適用してからファイルに書く**。フィルタリングせずにそのまま保存しない。
- **SNS投稿ネタ候補のフック候補は、アカウントのトーンに合わせる**。threads-draftやx-draftにそのまま渡せるクオリティで書く。
- **投稿タイミングを必ず明記する**。ニュースは鮮度が命なので、いつまでに投稿すべきかを判断する。

## 関連

- 出力先: `SNS/{アカウント名}/最新ニュース/`
- SNS投稿に使う場合は /threads-draft / /x-draft / /fb-draft に渡す
- アカウント設計: `SNS/{アカウント名}/アカウント設計.md`
- 投稿ルール: 各スキル（/threads-draft, /x-draft, /fb-draft）のSKILL.md内に定義
- 海外事例の深掘りは /threads-global-research を使う（本スキルとは棲み分け）
