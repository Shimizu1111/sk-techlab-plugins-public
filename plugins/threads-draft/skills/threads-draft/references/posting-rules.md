## 投稿ルール

### 最上位原則: すべり台（Slippery Slide）

**すべての文の唯一の目的は、次の文を読ませること。**

コピーライターJoseph Sugarmanの原則。投稿の1行目は2行目を読ませるために存在し、2行目は3行目を読ませるために存在する。最後の文に辿り着くまで、読者をすべり台のように滑らせ続ける。

#### すべり台を作る5つの武器

**1. オープンループ（未完了の問い）**
答えを言わずに問いだけ開く。人間の脳は開いたループを閉じたがる。

```
× 月次レポートをAIで自動化したら8時間が30分になりました。
  → 1文目で完結。2文目を読む理由がない

○ 毎月15日になると胃が痛くなる経営者、多いと思います。
  → 「なぜ？」が開く。閉じるまで読むしかない
```

使いどころ:
- 1投稿目の冒頭（最重要）
- 1投稿目の末尾（2投稿目へ引っ張る）
- 体験談の途中（「ある日、試しにやってみたんです。」で止める）

**2. バケツリレー接続（Bucket Brigade）**
文と文の間に短い橋渡しフレーズを挟んで、読者を次の文に運ぶ。

よく使うバケツリレー:
- 「でも、それだけじゃない。」
- 「ここからが本題。」
- 「正直に言うと——」
- 「しかし」「ところが」
- 「じゃあどうするか？」
- 「結論から言うと」

**3. 好奇心の種（Seeds of Curiosity）**
段落の終わりに、次を読みたくなる短い一文を仕込む。

テンプレート:
- 「結果は——」
- 「でも、問題はそこじゃなかった。」
- 「ここからが想定外だった。」
- 「理由は1つだけ。」
- 「で、どうなったか。」

**4. 常識の否定**
読者が「そうだよね」と思っていることの逆を言う。脳が「え？」となった瞬間、次を読まずにいられない。

**5. 情報の出し惜しみ**
核心を1投稿目で全部言わない。最も重要な情報ほど後ろに持っていく。

```
× 1投稿目: Claude Codeで月次レポートを自動化する方法を教えます。CSVを渡して...
  → 全部わかった。2投稿目いらない

○ 1投稿目: 毎月丸一日かけてた作業が、ある方法で30分になった。
  2投稿目: その方法は...
  → 「ある方法」が何かを知るには2投稿目を読むしかない
```

---

### 「それで？」テスト

**投稿の各文を読んだ後、読者が「それで？」「だから何？」と思ったら、その文は失敗。**

各文を読んだ後に読者の頭に浮かぶべきは:
- 「え、なんで？」（好奇心）
- 「マジで？」（驚き）
- 「それ俺のことじゃん」（共感）
- 「で、どうなったの？」（続きへの期待）

投稿を書いたら、1文ずつ指で隠しながら「次の文を読みたいか？」をチェックする。読みたくならない文があったら、その文を書き直すか削る。

### 構成フレームワーク

#### PAS（Problem → Agitation → Solution）
Threads/Xで最も効果的。中立フックの2倍のクリック率。

```
P: 毎月15日になると胃が痛くなる経営者、多いと思います。
A: 経理が丸一日かけて数字集めて、翌日グラフ作って、差し戻しが入って...
S: Claude Codeに投げたら30分で終わった。
```

#### BAB（Before → After → Bridge）
変身ストーリー型。Before/Afterの数字が強い時に使う。

```
B: 去年まで、月次レポートに2日かかっていた。
A: 今は30分。しかも人間より正確。
Bridge: やったことは1つだけ。Claude Codeにデータを渡しただけ。
```

#### ストーリー型（日常 → 事件 → 教訓）
体験談ベース。最も共感を生む。「過去の自分 ＝ 読者の今」として語る。読者が主人公に自分を重ねられる。

---

### スレッドの構成

```
1投稿目（フック → すべり台 → 好奇心の種）: 200〜450文字
  ↓ セルフリプライ
2投稿目（核心の回答 → 展開 → 余白）: 130〜450文字
  ↓ セルフリプライ（必要に応じて）
3投稿目（教訓 or CTA or 問いかけ）: 100〜300文字
```

- スレッドは2〜3投稿。10本も15本もセルフリプライするスタイルではない
- 1投稿 = 200〜450文字のしっかりした文章。細切れの1文ずつではない。1投稿の中に複数段落を含む

#### 各投稿の役割

**1投稿目: 止める → 引き込む → 2投稿目へ飛ばす**
- 冒頭2行でスクロールを止める（10〜20文字で判断される）
- すべり台で最後まで読ませる
- 末尾にオープンループか好奇心の種を置いて2投稿目へ

**2投稿目: 1投稿目のループを閉じる → 新しい価値を出す**
- 1投稿目で開いたオープンループの答えを出す
- ここで核心の情報・数字・ノウハウを出す
- 全部語り切らず「余白」を残す

**3投稿目: 読者に渡す**
- 問いかけ、CTA、または読者が自分に当てはめて考える余地を作る

---

### 改行スタイル（意味ブロック方式）

**同じ文脈の文は繋げ、文脈が変わるところで空行を入れる。**

- **同じ文脈** → 改行のみ（空行なし）で繋げる
- **文脈の転換** → 空行（1行空け）で区切る
- 「。」のたびに必ず空行を入れる必要はない。文脈が同じなら繋げた方が読みやすい
- 文脈の切れ目を正しく空けていれば、重要な1文は自然と孤立して目立つ

**実例（@yoshiaki_uchiyama_ / 4.3Kフォロワー）:**
```
「頑張って投稿してるのに売上が変わらない」

これ、めちゃくちゃ多いです。

でも正直に言うと              ← 同じ文脈だから改行のみ
原因は集客じゃないです。

売れない理由は                ← 文脈が変わったから空行
ほぼ100% 導線です。

・フォロワー増えてるのに来店ゼロ
・LINE登録あるのに予約入らない  ← リストは改行のみで並べる
・安いメニューしか売れない

これ全部                      ← 文脈が変わったから空行
導線設計ミスです。
```

### フックのルール

全ての投稿で最も重要。ここでスクロールが止まらなければ終わり。
**人は冒頭10〜20文字だけを見て、読む/読まないを判断する。**

#### フックの型

| 型 | 例 | なぜ機能するか |
|---|---|---|
| 恐怖+好奇心 | 「付き合ってもメリットのない、逆に損する○○がいます」 | 「誰？うちの担当は大丈夫？」→自分ごと化+オープンループ |
| 自己否定 | 「年収1億ある。しかし、私の価値は2028年激減する」 | 権威→否定の落差。「なぜ？」が強烈に開く |
| 常識の否定 | 「○○、ほぼ全部間違ってます」 | 「え、マジで？」脳が反射的に次を読む |
| 数字+リスト予告 | 「実際に自動化した業務5つ」 | 具体的な数字は抽象より3倍注目される |
| 体験談導入 | 「AIに任せて一番ミスった話」 | 失敗談は成功談より信頼される+好奇心 |
| 問いかけ（共感型） | 「頑張って投稿してるのに売上が変わらない」 | 「それ俺のことじゃん」→自分ごと化 |

#### フック後の展開パターン

フックで止めた後、すべり台に乗せる流れ:

**パターンA: 共感 → 否定 → 核心**
```
「頑張って投稿してるのに売上が変わらない」  ← 共感で止める

これ、めちゃくちゃ多いです。                ← 「自分だけじゃない」安心

でも正直に言うと                            ← バケツリレー
原因は集客じゃないです。                    ← 常識の否定。「え？じゃあ何？」

売れない理由は
ほぼ100% 導線です。                         ← 核心。でもまだ「どうすれば？」が残る
```

**パターンB: 権威 → 自己否定 → 理由は次へ**
```
顧問だけでも、年収1億あります。             ← 権威で止める

しかし、私の価値は、                         ← 「しかし」で急転換
2028年激減します。                           ← 自己否定。「なんで？」が爆発

（2投稿目で理由を語る）                      ← オープンループで引っ張る
```

**パターンC: 恐怖 → 具体化 → 判断基準**
```
付き合ってもメリットのない、逆に損する       ← 恐怖で止める
信用金庫の営業マンがいます。                 ← 「誰？」オープンループ

融資を餌にいろんなものを売りつけてきたり     ← 具体化。「あるある」
して、逆に資金繰りが悪化するケースも。       ← 恐怖の増幅

だからこそ見極めてください。                 ← 判断基準へ
```

#### 絶対やらない冒頭
- 「こんにちは」「おはようございます」
- 「今日も投稿します」
- 前置きや説明から入る
- **1行目で結論を全部言い切る**（すべり台が始まらない）

---

### 最終投稿のルール

3パターンから選ぶ:
1. **問いかけ**: 「あなたの会社だと何が自動化できそうですか？」
2. **フォローCTA**: 「こういうAI活用の実例、今後も出していくのでフォローしておいてください」
3. **リンク誘導**: 「詳細はプロフィールのリンクから」

**体験会・セミナー・イベントへの誘導は、ユーザーから明示的に指示がない限り絶対に書かない**（「プロフィールからどうぞ」等も不可）

---

### フォーマットルール

- **タグ**: つけない。伸びているアカウント全員ほぼ使っていない
- **外部リンク**: 本文に貼らない。プロフィール or 固定投稿に誘導
- **画像**: テキスト+画像のハイブリッドが強い。画像だけは弱い
- **絵文字**: 控えめ。使うとしてもポイント的に
- **トーン**: ベースは「です・ます」。断定やインパクトを出したい要所だけ「だ・である」を使う。全体としてちゃんとした言葉遣いを保つ（セミナー集客の導線であり、この人に会いたいと思わせる信頼感が必要）。「笑」は文脈次第でOK、「ww」は使わない
- **命令口調の禁止**: 「しろ」「するな」等の命令形は使わない。「してください」「してみてください」は可。断定調（「〜です」「〜じゃない」）はOKだが、読者への命令は炎上リスクがある

---

### 暗黙的ターゲティング（最重要ルール）

- 「経営者向け」「社長さんへ」とは絶対に書かない
- **経営者しか興味がない話題を自然に扱う**
- AI将軍方式: 「社員の仕事が3時間短縮」「月次レポートが自動化」「採用コスト削減」→ 経営者が勝手に反応する

---

### 経営者に刺さる内容の原則

経営者の関心は **売上・コスト・人材・時間** の4つだけ。

- 「新機能が出た」→「請求書が5分で終わる」
- 「プロンプト」→「部下への指示と同じ」
- 「API連携」→「社員を増やさずに回せる」

**反応するもの:**
- 数字のBefore/After
- 失敗談（成功より信頼される）
- コスト削減の具体額
- 「競合はもう始めてる」（FOMO）
- 「人を増やさずに回す」

**反応しないもの:**
- 技術の仕組み説明
- 統計データだけの投稿
- 「簡単です」の連発
- 抽象的なモチベーション論

---

### 頻度・タイミング

- 1日1〜3投稿
- 最低ライン: 週3回
- 最適時間帯: 平日朝7〜9時 / 昼12〜14時 / 夜21〜24時
- 火〜木がベスト
- **投稿する時間帯 = 投稿後1時間リプライに返信できる時間帯**

---

### アルゴリズムの核心

- Threadsは「会話量」で評価する。いいね数よりリプライの数と速度
- 投稿後10〜90分の初速が全て
- Threads全体の閲覧の50%が「リプライ」由来
- **投稿後1時間はThreadsに留まり、来たリプライ全てに返信する**
- 意見が分かれる問いかけ＝リプライが増える＝アルゴリズムに好まれる

---

### やってはいけないこと

- 「いいねしてね」「フォローしてね」の直接依頼（ペナルティ）
- AI生成文を無加工で大量投稿（BAN対象）
- 短時間の大量フォロー・アンフォロー
- 挨拶から始める
- 記事っぽい丁寧な構成にする
- 全部語りきって余白をゼロにする
- 1文ずつ細切れにスレッドを連ねる（鬱陶しいだけ）
- **1行目で結論を全部出す（すべり台が始まらない）**
- **バケツリレーなしで文を並べる（読者が途中で落ちる）**

---

### 海外事例・ソース記事の引用戦略

海外事例（`海外事例/` フォルダ）やソース記事を元に投稿する場合、ソースの出し方を**ネタの種類で使い分ける**。

#### A. 権威を借りる（ソース1つだけ明示）

主語が1つで強い時に使う。「開発者本人が〜」「30人の会社が〜」「アメリカの○○社が〜」等。

```
○ 「Claude Codeを作った本人が、自分の設定ファイルを公開した。驚いたのは"たった100行"だったこと。」
→ 権威ある1ソースを冒頭に。以降の解釈・翻訳は自分の見解として語る
```

向いているネタ: 開発者インタビュー、特定企業の導入事例

#### B. 自分の実感として語る（ソース出さない）

一般的なコツ・フレームワーク系。毎日使っている実践者として語る。

```
○ 「AIへの指示、"いい感じにやって"だと100%失敗する。部下と同じで"こうなったら完成"を伝えるだけで全然違う。」
→ 複数ソースの知見を自分のフィルターで消化した"自分の言葉"
```

向いているネタ: ノウハウ・Tips、使い方のコツ、フレームワーク

#### C. 「海外では〜」で匂わせる（ソース曖昧）

具体的な数字・金額がある事例。新鮮さ・先行事例感を出す。

```
○ 「アメリカでは外注費6種類を丸ごとAIに置き換えた会社が出てきてる。」
→ URLは出さない。興味を持った人がDM/リプで聞いてくる導線になる
```

向いているネタ: 海外企業の数字付き事例、コスト削減・売上向上の具体例

#### 判断基準

| ネタの特徴 | パターン | 理由 |
|---|---|---|
| 特定の人物・企業名が主語 | A（権威を借りる） | 「誰が」が明確な方がフックが強い |
| 自分も日常的にやっていること | B（自分の実感） | 実践者ポジションが強化される |
| 具体的な数字・金額がある海外事例 | C（海外匂わせ） | 数字のインパクト + 「日本ではまだ」感 |
| 複数ソースの横断知見 | B（自分の実感） | 出典を並べると学術論文っぽくなる |

#### 絶対やらないこと

- URLを本文に貼る（アルゴリズムペナルティ + 誰も踏まない）
- 複数ソースを列挙する（「○○社と△△社の調査によると〜」は記事調すぎ）
- 翻訳記事のまとめっぽくする（「海外ニュースまとめ」感は実践者ポジションと矛盾）
