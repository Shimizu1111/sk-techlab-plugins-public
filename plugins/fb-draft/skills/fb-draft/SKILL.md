# /fb-draft - Facebook投稿下書き生成

投稿ネタからFacebook投稿ルールに基づいた下書きを生成し、選択→確定→投稿済み管理まで行う。

**前提: Facebook個人アカウント（実名: 清水勝紀）。Threads/Xの「AI将軍」と同一人物の実名投稿。**


**投稿ルールの詳細は `references/posting-rules.md` を参照。下書き生成前に必ず読み込むこと。**

## 引数

- `$ARGUMENTS` にテーマやファイルパスを渡せる（省略可）

## 実行手順

### Step 0: アカウント選択

`SNS/` 配下のフォルダ一覧を取得し、ユーザーに選んでもらう。
`$ARGUMENTS` にアカウント名やファイルパスが含まれていれば自動で特定してスキップ。

### Step 1: ルール読み込み

選択されたアカウントの以下を**すべて読み込む**:

1. **`自社情報.md`** — 事業内容・経歴・クライアント事例・AI活用実態。**投稿の具体性・リアリティの源泉**
2. **`アカウント設計.md`** — ターゲット・ポジショニング・トーン
3. **`投稿案_*.md`** — 既存投稿案があればトーンの参考として直近1ファイル

**読み込んだルールは以降のすべての下書き生成で厳密に適用する。**

### Step 2: 投稿ネタ選択

`$ARGUMENTS` でファイルパスが指定されていればそのファイルを使う。

指定がなければ:
1. `SNS/{アカウント名}/投稿ネタ/` 配下のファイルを一覧表示する
2. 各ファイルのフロントマター（status, priority）を読み取り、一覧に表示する
3. `status: idea` のものを優先的に上に表示する
4. ユーザーに番号で選んでもらう（複数選択も可）

### Step 3: 投稿形式と画像の判断

Facebookは基本**単一投稿で完結**。ネタの内容に応じて最適な文字数帯を提案する:

| 形式 | 向いているネタ | サイズ |
|---|---|---|
| ショート投稿 | 一言の気づき・共感・問いかけ | 150〜300文字 |
| スタンダード投稿 | 体験談・ノウハウ・事例紹介 | 500〜800文字（推奨） |
| ロング投稿 | ステップ解説・詳細事例・チェックリスト | 800〜1200文字 |

ユーザーに形式を確認し、希望があれば変更する。

#### 画像の判断

**毎回画像をつけない。** 毎回きれいな画像がつくと「ブランドアカウント」感が出て知人からの親近感が減る。テキストだけの投稿が混ざることで人間味が出る。**3回に1〜2回くらい画像あり**がバランスとして良い。

ネタの内容に応じて画像の有無を提案し、ユーザーに確認する:

**画像あった方がいいケース:**
- 数字のBefore/After系 → 視覚的な比較が刺さる
- ノウハウ・手順系 → 要点をまとめた画像は保存されやすい
- 長文（600文字超） → 画像がフィードで目を止める役割になる

**なくていい（むしろない方がいい）ケース:**
- 短い気づき・問いかけ系 → テキストだけの方が「ふと思ったこと」感が出て自然
- 日常の共有系 → 画像つけると「作り込んでる」感が出る
- 議論を呼びたい投稿 → テキストだけの方がコメントに集中しやすい

**画像を使う場合の種類:**

| 種類 | 備考 |
|---|---|
| リアル写真（作業風景・スクショ） | **最も信頼感が高い。推奨** |
| テキスト画像（Before/After、数字まとめ） | 保存されやすい。AI臭くない |
| ChatGPT GPT Image 2.0で生成 | 写実的で高品質。テキスト確定後にプロンプトを出力する |

画像ありの場合は、テキスト確定後にStep 6.5で画像プロンプトを生成する。

### Step 4: 下書き生成（複数パターン）

選択されたネタファイルを読み込み、**2パターンの下書き**を生成する。
各パターンは異なるフックの型と構成パターンを使い、切り口を変える。

**`references/posting-rules.md` の全ルールに従って下書きを生成する。** 必ず事前に読み込むこと。

### Step 5: パターン提示

```
## パターンA: {構成パターン名}（{フックの型}）

**文字数:** {文字数}文字
**「もっと見る」前（{文字数}文字 / {行数}行）:**

{フック部分のテキスト}

──── もっと見る ────

{本文の続き}

{CTA}

**引き込み分析:** {使用した武器}
**保存・シェア狙い:** {保存/シェアされそうな要素}

---

## パターンB: {構成パターン名}（{フックの型}）
...
```

提示後、ユーザーに聞く:
**「どのパターンがいい？ そのまま使う or 修正したい部分ある？」**

### Step 6: フィードバック対応

ユーザーからのフィードバックに応じて修正する。

- 「もっとカジュアルに」「もっと丁寧に」等のトーン調整
- フックの差し替え
- 文字数の調整
- 内容の追加・削除
- パターンの組み合わせ（「Aのフック + Bの本文」等）

納得いくまで何度でも修正する。

### Step 6.5: 画像プロンプト生成（画像ありの場合のみ）

Step 3で画像ありと決まった場合、テキスト確定後に実行する。

**画像生成はChatGPT GPT Image 2.0を使う（Claude Codeではやらない）。**
**Claude CodeからCodexエージェントを立ち上げて画像生成を実行する。**

**Claude Code側では画像の内容（何を伝えたいか）だけを指定する。デザイン・スタイル・色・フォント等の指定はしない。** ChatGPT側の画像生成の方がデザイン判断は優秀なので、そこは任せる。

投稿テキストの核心メッセージから画像の「内容指示」を生成し、Bashツールで `codex exec` コマンドを実行して画像を生成する:

```bash
codex exec "以下の内容でFacebook投稿用の画像を1枚生成してください。サイズは1200x630px。保存先: {保存パス}

{内容指示（何を伝えたいか・画像に含める要素のみ）}"
```

Codexが使えない場合はプロンプトをユーザーに提示し、ChatGPTで手動実行してもらう。

**画像プロンプトの方針:**
- **内容だけ指定する**（伝えたいメッセージ、画像に含めるテキスト・数字・要素）
- **デザインは指定しない**（色、フォント、レイアウト、スタイル等はChatGPTに任せる）
- サイズのみ指定: 1200x630px（Facebook推奨）

### Step 7: 投稿確定

ユーザーが「OK」「これでいこう」「投稿する」等と言ったら:

**「この内容で投稿済みにしていい？」** と最終確認する。

確認が取れたら以下を実行:

1. **投稿案ファイルに保存**
   - 保存先: `SNS/{アカウント名}/投稿案_{YYYYMMDD}.md`
   - 同じ日付のファイルが既にある場合は末尾に追記する

2. **元の投稿ネタファイルのステータスを更新**
   - `status: idea` → `status: posted`
   - `posted_date:` → `posted_date: {YYYY-MM-DD}`
   - `platform: Facebook` を追加

3. **投稿済みフォルダに移動**
   - 移動先: `SNS/{アカウント名}/投稿済み/`（フォルダがなければ作成）
   - 元の `投稿ネタ/` から `投稿済み/` にファイルを移動する

4. **投稿時のリマインド**
   - **「投稿後1時間以内に来たコメントに必ず返信してください（コメント対話がアルゴリズム最重要シグナル）」**
   - **「リンクがある場合はコメント欄に追加してください」**
   - 推奨投稿時間帯: 火〜木の20:00〜21:00（経営者層の黄金時間帯）
   - 残りの未投稿ネタ（`status: idea`）の件数

### 投稿案ファイルのフォーマット

```markdown
---

## 投稿{N}【Facebook】: {構成パターン名}（元ネタ: {ネタファイル名}）

**形式:** {ショート / スタンダード / ロング}
**文字数:** {文字数}文字
**引き込み:** {使用した武器}

**投稿:**
{本文}

**コメント欄に追加（リンクがある場合）:**
{リンクURL}
```

### 投稿前チェック（下書き生成時に自動で確認）

#### 「もっと見る」チェック（最優先）
- [ ] 冒頭2〜3文が短く切れているか？（1文10〜20文字。長い1文で始めていないか）
- [ ] 最初の150文字（5行以内）で「読む価値がある」と感じるか？
- [ ] 引きが強すぎないか？（オープンループ・煽り・出し惜しみを使っていないか）
- [ ] 「もっと見る」をタップしたくなるか？
- [ ] フック部分で結論を全部言い切っていないか？
- [ ] PC版で5行以内に収まっているか？（空行も1行カウント）

#### 構成チェック
- [ ] PAS / BAB / ストーリー型のいずれかの構成か？
- [ ] 500〜800文字に収まっているか？
- [ ] 段落ブロック（2〜3文ごとに改行）で読みやすいか？
- [ ] バケツリレー（接続フレーズ）で段落がつながっているか？

#### コンテンツチェック
- [ ] です・ます調で統一されているか？（時々カジュアル混在はOK）
- [ ] クライアントの事例・エピソードを使っていないか？（特定リスク）
- [ ] 末尾に問いかけCTAがあるか？（エンゲージメントベイトでないこと）
- [ ] 経営者の関心（売上・コスト・人材・時間）に着地しているか？
- [ ] 専門用語を使っていないか？
- [ ] 80/20ルール: 宣伝になっていないか？
- [ ] 保存・シェアしたくなる内容か？
- [ ] 「過去の自分＝読者の今」として語れているか？
- [ ] 人間味があるか？（AI臭くないか）
- [ ] 営業感が出ていないか？（知人が読む前提）

#### フォーマットチェック
- [ ] リンクが本文にないか？（コメント欄に置く）
- [ ] ハッシュタグは1個以下か？

#### 炎上チェック（Facebook実名チェック）
- [ ] 実名で投稿して恥ずかしくない内容か？
- [ ] 友人・取引先が見ても問題ない表現か？
- [ ] 他者・他サービスを否定していないか？
- [ ] 読者を見下す表現になっていないか？
- [ ] 過激な表現がないか？（Threads/Xより抑えめに）

#### アルゴリズムチェック
- [ ] 投稿後1時間以内にコメント返信できる時間帯か？
- [ ] エンゲージメントベイト（「いいねして」系）になっていないか？

## 使用例

```
/fb-draft
→ アカウント選択 → ネタ一覧 → 形式提案 → 複数パターン生成 → 選択 → 投稿済み

/fb-draft SNS/AI将軍/投稿ネタ/260525-ChatGPTとClaudeCodeの違い.md
→ 指定ファイルから直接下書き生成

/fb-draft AI将軍
→ アカウント指定 → ネタ一覧 → 選択 → 下書き生成
```
