## 投稿ルール

### 最上位原則: 信頼の長文（Trust Long-form）

**実名の信頼を武器に、体験・ノウハウ・ストーリーを500〜800文字で届ける。**

Facebookは実名制。Threads/Xの匿名バズとは違い、「この人が言うなら」の信頼が効く。
長文が読まれる唯一のSNSであり、体験と具体性で勝負する。

### Threads/Xとの最大の違い

| 観点 | Threads | X（無料） | **Facebook** |
|---|---|---|---|
| 文字数 | 500文字 | 140文字 | **63,206文字（実質500〜800文字推奨）** |
| 名前 | AI将軍 | AI将軍 | **清水勝紀（実名）** |
| 最上位原則 | すべり台 | 密度弾 | **信頼の長文（Trust Long-form）** |
| フォーマット | スレッド2〜3投稿 | 単発140文字 | **単一投稿で完結** |
| 改行 | 1文1ブロック | 面積支配 | **段落ブロック（2〜3文で1段落）** |
| 最強シグナル | 会話量（リプ） | リプ連鎖(x150) | **保存 > DM共有 > コメント対話** |
| リンク | 本文NG | 本文NG | **本文NG（月2件制限。コメント欄に置く）** |
| トーン | カジュアル断定 | カジュアル断定 | **実名の信頼感+断定。少し丁寧寄り** |
| 投稿頻度 | 1日1〜3 | 1日1〜3 | **1日1投稿（週3〜7回）** |
| ハッシュタグ | つけない | つけない | **最大1個（基本つけない）** |

#### Threads/Xの技術をFacebookに変換する

| Threads/Xの武器 | Facebookでの変換 |
|---|---|
| すべり台 / 密度弾 | → **「もっと見る」の壁**: 最初の150文字（5行）で引き込み、続きを読ませる |
| オープンループ | → **そのまま使える**: 冒頭で問いを開き、本文で回答する。長文だから1投稿で完結できる |
| バケツリレー | → **段落のつなぎに活用**: 長文でも離脱させないブリッジフレーズ |
| 面積支配 | → **段落ブロック**: 2〜3文で改行。壁テキストを防ぐ |
| 余白の引力 | → **問いかけCTA**: 末尾で「あなたの場合は？」と問い、コメントを誘発 |
| 情報の出し惜しみ | → **使いすぎない**: Facebookは価値提供の完結感が重要。出し惜しみしすぎると信頼を損なう |

### 「もっと見る」の壁（最重要）

Facebookの生命線。ここを突破できなければ投稿は読まれない。

#### 表示ルール

| デバイス | 折りたたみ基準 |
|---|---|
| PC | **6行目から**省略（行数判定。空行も1行カウント） |
| スマホ | **150文字目から**省略（文字数判定。改行も1文字カウント） |
| 両方で全文表示 | **改行含め150文字以内 かつ 5行以内** |

#### 「もっと見る」の前に仕込むこと

**最初の150文字（5行以内）= フック+続きを読む理由。**

```
○ 良い例:
月次レポート、去年まで丸2日かかっていた。
今は30分。しかも人間より正確。

やったことは1つだけ。（← ここで「もっと見る」）

× 悪い例:
今日はAIについて書きたいと思います。
最近Claude Codeというツールを使い始めたのですが、
これがなかなか面白くて...（← 何も引っかからない）
```

**テクニック:**
- 数字のBefore/Afterを冒頭に置く
- 結論を匂わせて「もっと見る」の向こうに本文を配置
- 問いかけで「自分のことだ」と思わせる
- 「もっと見る」の直前で文を途切れさせる

### 投稿の構成フレームワーク

#### PAS（Problem → Agitation → Solution）+ 問いかけ

Facebookで最も効果的。長文の余裕を活かして各パートを厚く書ける。

```
【フック（150文字以内）】
毎月15日になると胃が痛い。
月次レポートの締め日。経理が丸一日かけて数字を集めて、翌日グラフを作って、
差し戻しが入って...この光景、毎月繰り返していませんか。

──── もっと見る ────

【Agitation（共感の深掘り）】
正直、自分もそうだった。
「これどうにかならないか」と思いながら、でも属人化してて
誰にも引き継げない。外注するほどの量でもない。
結局、毎月同じ苦痛を繰り返していた。

【Solution（解決策）】
ある日、試しにClaude Codeにデータを渡してみた。
CSVを投げて「月次レポートの形式で整理して」と伝えただけ。
30分で終わった。精度は体感90%以上。

【教訓・着地】
技術の問題じゃなかった。
「これ、AIに任せられるんじゃないか？」と気づくかどうか。
それだけの話だった。

【CTA】
皆さんの会社で「毎月これ面倒だな」と思ってる作業、何がありますか？
```

#### BAB（Before → After → Bridge）

数字が強い時に使う。

```
【フック】
請求書作成、毎月3時間かけていた。
今は「先月の請求書作って」の一言で5分。

──── もっと見る ────

【Before → After の詳細】
去年まで:
・Excelでテンプレ開いて手入力
・宛先と金額を毎回確認
・上長に回して修正が戻ってきて...

今:
・Claude Codeにデータを渡す
・「前月と同じ形式で」と言うだけ
・チェックして送るだけ

【Bridge】
特別なことはしてない。
「繰り返してる作業」をAIに渡しただけ。

【CTA】
最初にAIに任せるなら、こういう"地味だけど毎月やってる作業"がおすすめです。
何から始めたらいいか迷ってる方、コメントで聞いてください。
```

#### ストーリー型（日常 → 事件 → 教訓）

最も保存・シェアされやすい。

```
【フック】
社員5人の会社の社長にAIを見せた時の話。
「パソコン苦手です」と最初に言われた。

──── もっと見る ────

【ストーリー展開】
正直、この時点で「難しいかな」と思った。
でもとりあえず目の前でやってみた。

議事録のテキストを渡して
「来週までのアクションだけ抜き出して」と言った。
30秒で一覧が出てきた。

社長の表情が変わった。
「これ、毎週やってたやつじゃん」

【教訓】
AIの導入に必要なのは、技術力じゃない。
「この作業、もっと楽にならないかな」という発想だけ。

パソコン苦手な社長でも、
「何を楽にしたいか」は誰より知っている。
それがAI活用の出発点だった。

【CTA】
「うちの会社でもできるかな？」と思った方、
気軽にコメントやメッセージください。
```

### トーンのルール

#### Facebook固有のトーン設計

Threads/X（AI将軍）とFacebook（清水勝紀）のトーンは**同じ人間の別の顔**。

| 要素 | Threads/X（AI将軍） | Facebook（清水勝紀） |
|---|---|---|
| 一人称 | 自分 / 省略 | **自分 / 私** |
| 語尾 | 断定調。時々カジュアル | **です・ます調ベース。時々カジュアル混在OK** |
| 専門用語 | 噛み砕く | **噛み砕く（同じ）** |
| 人間味 | 投稿内容で示す | **仕事の裏側・日常・失敗談も混ぜる** |
| 営業感 | 出さない | **出さない（同じ。80/20ルール）** |
| 信頼の源泉 | 投稿の質 | **実名+投稿の質+友人・知人の繋がり** |
| テクニック | すべり台・オープンループ等 | **使わない。友人への自然な共有** |
| 事例 | クライアント事例OK | **クライアント事例は使わない（特定リスク）** |

#### 知人向けトーンの原則（最重要）

**Facebookの友人層はBNI経由の経営者や事業者。知り合いが読む前提。**

- Threads/Xは見知らぬ人のスクロールを止める必要がある → テクニックが必要
- Facebookは「清水さんの投稿だ」で一定の信頼がある → **自然に価値ある話題を置くだけでいい**
- 引きが強すぎると「営業っぽい」「情報商材っぽい」印象を与え**逆効果**
- オープンループ・煽り・出し惜しみは知人には一発で見透かされる
- **クライアントの事例・エピソードは使わない**（知人が多いため特定されるリスク）
- **自分自身の体験・気づき・実務を中心に書く**

#### Threads/Xとの使い分け

- **同じネタをFacebook用にリライトする**のが効率的
- Threadsで断定調で書いたものを、Facebookでは「体験談」として丁寧に展開する
- Facebookの方が長文OKなので、背景やストーリーを厚く書ける
- ただし、全く同じ文面のマルチポストは避ける（各プラットフォームの空気に合わせる）
- **Threads/Xのテクニック（オープンループ等）はFacebookに持ち込まない**

### 改行スタイル

Facebookは段落ブロック式。Threads（1文1ブロック）ともX（面積支配）とも違う。

#### 基本: 2〜3文で1段落

```
月次レポート、去年まで丸2日かかっていた。
経理が数字を集めて、グラフを作って、差し戻しが入って。
毎月同じことの繰り返しだった。

ある日、試しにClaude Codeにデータを渡してみた。
CSVを投げて「月次レポートの形式で整理して」と伝えただけ。

30分で終わった。

技術の問題じゃなかった。
「これ、AIに任せられるんじゃないか？」と気づくかどうか。
それだけの話。
```

#### 空行と改行の使い分け（意味ブロック方式）

- **空行（1行空け）= 文脈の切り替わり:**
  - 話題・文脈が変わるところ
  - 読者に考える時間を与えたいとき
  - 結論や重要な一文を際立たせたいとき（ただし多用しない）
- **改行のみ（空行なし）= 同じ文脈の中:**
  - 同じ文脈の文は繋げる（「。」のたびに必ず空行を入れる必要はない）
  - テンポよく畳み掛けたいとき
  - 事実の列挙

段落ブロック（2〜3文で1段落）のルールと自然に一致する。同じ文脈の文を2〜3文まとめて1段落とし、文脈が変わるところで空行を入れる。

#### PC版の注意

PCでは**空行も1行としてカウント**されるため、空行を入れすぎると「もっと見る」が早く出る。
フック部分（最初の5行）では空行は最小限にする。

```
○ フック部分（PC対策: 5行以内に収める）:
月次レポート、去年まで丸2日かかっていた。
今は30分。しかも人間より正確。
やったことは1つだけ。
                         ← ここで「もっと見る」（3行で収めた）

× フック部分がPC版で途切れる例:
月次レポート。

去年まで丸2日かかっていた。

今は30分。
                         ← PC版ではここでもう5行（空行含む）。「もっと見る」に
```

### フック（冒頭150文字）のルール

#### フックの型

| 型 | Facebook用の実例 | なぜ機能するか |
|---|---|---|
| 恐怖+好奇心 | 「AI導入の見積もり、中身を見ると驚く内容だった。」 | 実名で柔らかく書いても好奇心は開く |
| 体験談導入 | 「社員5人の会社の社長にAIを見せた時の話。」 | Facebookで最も強い型。ストーリーの入口 |
| Before/After | 「月次レポート、去年まで丸2日。今は30分。」 | 数字の落差で止める |
| 問いかけ（共感型） | 「毎月の締め日、経理に丸一日取られていませんか。」 | 「自分のことだ」→自分ごと化 |
| 常識の否定 | 「AIを導入するのに、技術力は必要なかった。」 | 「え？」で次を読みたくなる |
| ノウハウ予告 | 「Claude Codeで実際に自動化した業務を全部書きます。」 | 保存・シェア狙い。Facebookの強み |

#### 絶対やらない冒頭

- 「おはようございます」「こんにちは」
- 「今日は〜について書きます」
- 前置きや自己紹介から入る
- 1行目で結論を全部言い切る

### CTA（行動喚起）のルール

#### Facebookで効くCTAの型

**最強: コメントを促す問いかけ（アドバイス・体験を聞く形式）**

```
○ ペナルティなし（アドバイス・体験を聞く形式）:
「皆さんの会社で毎月面倒だと思ってる作業、何がありますか？」
「同じ経験された方いますか？どう対処しましたか？」
「AIで最初に自動化するなら何がいいと思いますか？」

× ペナルティあり（エンゲージメントベイト）:
「参考になったらいいねしてね」
「シェアしてください」
「Aだと思う人はいいね、Bだと思う人はコメント」
```

#### CTAの配置

- **短い投稿（150文字以下）**: 末尾に配置
- **長い投稿（150文字超）**: 「もっと見る」の**前にも**軽いフックCTA + 末尾に本CTAの二段構え
- 体験会・セミナー誘導は**ユーザーから明示的に指示がない限り書かない**

### リンクの扱い

- **本文にリンクを入れない**（月2件制限 + エンゲージメント最低）
- リンクが必要な場合は**コメント欄に自分で追加する**
- 本文では「詳しくはコメント欄にリンク置きました」と案内

### コンテンツ配分（80/20ルール）

| カテゴリ | 割合 | 内容 |
|---|---|---|
| 体験談・ストーリー | 40% | 「実際にやってみた」系。失敗談も含む |
| ノウハウ・Tips | 25% | 保存されやすい。チェックリスト・手順系 |
| 日常・人間味 | 15% | 仕事の裏側、気づき。AI臭くない投稿 |
| 宣伝・告知 | **20%以下** | 体験会案内など。5投稿に1回が上限 |

**80%は純粋な価値提供。宣伝は20%以下に抑える。**

### アルゴリズムの核心

#### 最重要シグナル（重要度順）

1. **保存（Save）**: 「後で見たい」= 高価値コンテンツの最強シグナル
2. **DM/Messenger共有**: 「友人に教えたい」= 究極の信頼票
3. **コメント対話**: 10件の深いコメント > 100件のいいね。往復が重要
4. **滞在時間・読了率**: じっくり読まれる長文投稿が高評価
5. **いいね・リアクション**: 最も弱いシグナル

#### 保存・シェアされるコンテンツの特徴

- **ノウハウ・チェックリスト型**: 「後で実践しよう」と保存される
- **体験ベースのストーリー**: 「知人に教えたい」とシェアされる
- **Before/Afterの数字**: 「うちの会社でもできるかも」と保存される
- **共感できる失敗談**: 「自分だけじゃないんだ」とシェアされる

#### コメント対話を促す投稿設計

- 末尾に問いかけ（アドバイス・体験を聞く形式）
- 意見が分かれるテーマ（ただし炎上しない範囲）
- **投稿後1時間はFacebookに留まり、来たコメントすべてに返信する**
- 1行の「ありがとうございます」ではなく、相手の内容に触れた返信をする

### フォーマットルール

| 項目 | ルール |
|---|---|
| ハッシュタグ | **基本つけない。つけても最大1個** |
| リンク | **本文に入れない。コメント欄に置く（月2件制限）** |
| 画像 | テキスト+画像がベスト。1200x630px推奨 |
| 絵文字 | **控えめ。1投稿1〜2個まで**。箇条書きの→やチェックはOK |
| トーン | **実名の信頼感+断定調。「です・ます」ベースに時々カジュアル** |
| 文字数 | **500〜800文字推奨。最低でもフック150文字** |

### 頻度・タイミング

- **推奨: 1日1投稿（週3〜7回）**
- 上限: 1日2投稿（それ以上はオーバーサチュレーション）
- **最適時間帯: 火〜木の20:00〜21:00**（日本の経営者層の黄金時間帯）
- 次点: 平日朝7〜9時 / 昼12〜13時
- 土日はエンゲージメント低い。避けるのが無難
- **投稿後1時間はFacebookに留まり、コメントに返信する**
- 一貫性が最重要: 不規則な投稿より定期投稿の方が5倍のエンゲージメント

### 炎上防止ルール

#### 3原則

1. **他者・他サービスを否定しない**
2. **読者を見下さない**
3. **属性を攻撃しない**

#### Facebook固有の注意（実名制のため影響が大きい）

- **実名で投稿している = 半永久的に本人と紐づく**
- 友人・知人・取引先が見ている前提で書く
- Threads/Xで許される強い表現もFacebookでは控えめにする
- 政治的・宗教的な話題には踏み込まない
- クライアント事例は必ず許可を取ってから
- 収入・売上の誇張表現は信頼を損なう
- 投稿を削除してもスクリーンショットが残るリスクを常に意識

#### Threads/Xとの表現トーン比較

```
Threads/X: 「AI導入の見積もり出してくる会社、9割ボッタクリです。」
Facebook:  「AI導入の見積もり、中身をよく見ると驚くことがある。」
           → 同じ内容だが、実名で攻撃的にならない表現に

Threads/X: 「知らない人は損してます。」
Facebook:  「知っておいて損はない話。」
           → 読者を見下さない柔らかい表現に
```

### やってはいけないこと

#### 共通NG

- 「いいねしてね」「シェアしてね」の直接依頼（ペナルティ）
- AI生成文を無加工で投稿（テンプレ感が出る）
- 挨拶から始める
- 記事っぽい丁寧な構成にする
- 全部語りきって余白をゼロにする
- 1行目で結論を全部出す

#### Facebook固有NG

- **リンクを本文に入れる**（月2件制限 + エンゲージメント最低）
- **毎回同じテンプレート構成**（AIが検知し配信制限）
- **ハッシュタグを3個以上つける**（エンゲージメント低下）
- **宣伝投稿を連投する**（80/20ルール違反。信頼残高が消える）
- **壁テキスト（改行なし長文）**（誰も読まない）
- **短時間の大量フレンド申請・グループ参加**（スパム判定）
- **同じ文面のコピペDM送信**（アカウント停止リスク）
- **過激な表現**（実名で半永久的に残る）

### 暗黙的ターゲティング（最重要）

- 「経営者向け」「社長さんへ」とは絶対に書かない
- **経営者しか興味がない話題を自然に扱う**
- 「月次レポートが自動化」「社員の仕事が3時間短縮」「採用コスト削減」→ 経営者が勝手に反応する

### 経営者に刺さる内容の原則

経営者の関心は **売上・コスト・人材・時間** の4つだけ。

#### 反応するもの
- 数字のBefore/After
- 失敗談（成功より信頼される）
- コスト削減の具体額
- 「競合はもう始めてる」（FOMO）
- 「人を増やさずに回す」

#### 反応しないもの
- 技術の仕組み説明
- 統計データだけの投稿
- 「簡単です」の連発
- 抽象的なモチベーション論
#### 「もっと見る」の壁の設計

**最初の150文字（5行以内）で「読む価値がある」と感じさせる。ただし煽りではなく自然に。**

**Facebookの原則: 知人の信頼 > テクニック**
- Threads/Xは見知らぬ人のスクロールを止める必要がある → テクニックが必要
- Facebookは「清水さんの投稿だ」で一定の信頼がある → 自然に価値ある話題を置くだけでいい
- 引きが強すぎると「営業っぽい」「情報商材っぽい」印象を与え逆効果

**冒頭2〜3文の設計:**
- 1文目は短く（10〜20文字）。内容自体が引きになるように
- 2文目も短く。1文目を補強 or 結果を示す
- 3文目以降から本文に自然に入る
- PC版: 空行も1行カウント → フック部分では空行を最小限に
- スマホ版: 150文字で折りたたみ → 150文字以内にフック完結

```
○ 良い設計（自然に引く）:
AIに自分の情報を全部渡してみました。
出てくる答えが、全然違いました。

何をしたかというと...

○ 良い設計（問いかけで止める）:
AIに何か聞くとき。
毎回同じ説明していませんか。

自分はずっとそうでした...

× 悪い設計（テクニック感が出る）:
やったことは1つだけ——
                         ← オープンループ。知人には「コピーライティングだな」とバレる

× 悪い設計（長い1文で始める）:
AIに毎回「うちの事業はこうで、今の課題はこれで...」って説明するのが面倒で、試しに...
                         ← 1文目が長すぎて流し読みされる

× 悪い設計（前置きから始める）:
今日はAI活用について書きます。
                         ← 何も伝わらないまま折りたたみ
```

#### トーンのキャリブレーション

Facebookは「友人への自然な共有」。Threads/Xのテクニックをそのまま持ち込まない。

```
○ Facebook（清水勝紀）のトーン:
「AIに自分の情報を全部渡してみました。出てくる答えが、全然違いました。」
「月額約3,000円なので、もっと早く気づきたかったです。」
「例えるなら、毎回初対面のコンサルに一から話すのと、5年付き合っている顧問に相談するくらいの差です。」

× Facebookに合わないトーン（Threads/Xの手法の持ち込み）:
「やったことは1つだけ——」（オープンループ。知人には営業感が出る）
「世界が変わった。」（煽りすぎ。友人に話すときにこうは言わない）

× Facebookに合わないトーン（その他）:
「9割ボッタクリです。」（実名で攻撃的すぎる）
「AIの活用により業務効率化が期待できます」（記事調すぎ）
「マジでヤバいww」（Facebookの空気に合わない）
「知らない人は全員バカです」（攻撃的。実名なので絶対NG）
```

### Facebook下書き生成の絶対ルール（Step 4用）

1. **です・ます調で書く**（清水勝紀のキャラに合わせる。時々カジュアルな混在はOK）
2. **引きを強くしすぎない**（オープンループ・煽り・出し惜しみテクニックは使わない。友人への自然な共有トーンで）
3. **冒頭に問いかけ型フックを使わない**（「〜していませんか？」「〜ではないですか？」等の読者への問いかけで始めない。知人には営業・セミナー導入に聞こえる。問いかけは末尾CTAでのみ使う）
4. **冒頭2〜3文は短く切る**（1文10〜20文字程度。短い文で自然に引く。長い1文で始めない）
5. **クライアントの事例・エピソードは使わない**（知人が多いFacebookでは特定されるリスクがある）
6. **自分自身の体験・気づき・実務を中心に書く**（自社情報.md のAI活用実態・経歴を参照）
7. **料金に触れるときは「読者が始められる金額」で書く**（自分のプランや料金は公開しない。「月額約3,000円で始められます」等。詳細は自社情報.md参照）
8. **炎上防止3原則を遵守する**（他者否定しない、読者を見下さない、属性を攻撃しない）
9. **実名チェック: 友人・取引先が見ても問題ない表現にする**
10. **ハッシュタグは基本つけない（つけても最大1個）**
11. **リンクは本文に入れない**（コメント欄に置く旨を案内する場合のみ記載）
12. **体験会・セミナー・イベントへの誘導は、ユーザーから明示的に指示がない限り絶対に書かない**
13. **改行は段落ブロック式**（2〜3文で1段落。壁テキスト厳禁）
14. **末尾に問いかけCTA**（コメントを促す。エンゲージメントベイトにしない）

### 海外事例・ソース記事の引用戦略

海外事例（`海外事例/` フォルダ）やソース記事を元に投稿する場合、ソースの出し方を**ネタの種類で使い分ける**。Facebookは実名投稿なので、特に「自分が実践者として語っている」感が重要。

#### A. 権威を借りる（ソース1つだけ明示）

主語が1つで強い時に使う。「開発者本人が〜」「30人の会社が〜」等。

```
○ 「Claude Codeを作った方が、自分の設定ファイルを公開していました。驚いたのは"たった100行"だったこと。」
→ 権威ある1ソースを自然に紹介。以降の解釈は自分の体験・見解として語る
```

向いているネタ: 開発者インタビュー、特定企業の導入事例

#### B. 自分の実感として語る（ソース出さない）

一般的なコツ・フレームワーク系。毎日使っている実践者として語る。**Facebookではこれが最も自然。**

```
○ 「AIに仕事を頼むとき、"いい感じにやって"だとまず期待通りにならないです。部下への指示と同じで、"こうなったら完成"を伝えるだけで精度が全然変わります。」
→ です・ます調で自然に。友人に話す感覚
```

向いているネタ: ノウハウ・Tips、使い方のコツ、フレームワーク

#### C. 「海外では〜」で匂わせる（ソース曖昧）

具体的な数字・金額がある事例。

```
○ 「最近知った話なんですが、アメリカの会社が外注していた業務6種類をAIで社内化して、年間数百万円の外注費がゼロになったそうです。」
→ 「最近知った話」で自然に導入。URLは出さない
```

向いているネタ: 海外企業の数字付き事例、コスト削減・売上向上の具体例

#### 判断基準

| ネタの特徴 | パターン |
|---|---|
| 特定の人物・企業名が主語 | A（権威を借りる） |
| 自分も日常的にやっていること | B（自分の実感）**← FB最多** |
| 具体的な数字・金額がある海外事例 | C（海外匂わせ） |
| 複数ソースの横断知見 | B（自分の実感） |

#### 絶対やらないこと

- URLを本文に貼る（コメント欄に置くルールと同じ）
- 複数ソースを列挙する（記事調すぎる）
- 翻訳記事のまとめっぽくする（知人に「業者っぽい」と思われる）
- 「海外ニュースまとめ」的な体裁（実名で自然に語る方が信頼される）
