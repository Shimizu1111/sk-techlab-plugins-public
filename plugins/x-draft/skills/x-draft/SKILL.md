# /x-draft - X投稿下書き生成

投稿ネタからX投稿ルールに基づいた下書きを生成し、選択→確定→投稿済み管理まで行う。

**前提: X無料プラン（1ツイート140文字制限）。ハッシュタグは使わない。**


**投稿ルールの詳細は `references/posting-rules.md` を参照。下書き生成前に必ず読み込むこと。**

## 引数

- `$ARGUMENTS` にテーマやファイルパスを渡せる（省略可）

## 実行手順

### Step 0: アカウント選択

`SNS/` 配下のフォルダ一覧を取得し、ユーザーに選んでもらう。
`$ARGUMENTS` にアカウント名やファイルパスが含まれていれば自動で特定してスキップ。

### Step 1: ルール読み込み

選択されたアカウントの以下を**すべて読み込む**:

1. **`アカウント設計.md`** — ターゲット・ポジショニング・トーン
2. **`投稿案_*.md`** — 既存投稿案があればトーンの参考として直近1ファイル

**読み込んだルールと上記「投稿ルール」セクションの全ルールを以降のすべての下書き生成で厳密に適用する。**

### Step 2: 投稿ネタ選択

`$ARGUMENTS` でファイルパスが指定されていればそのファイルを使う。

指定がなければ:
1. `SNS/{アカウント名}/投稿ネタ/` 配下のファイルを一覧表示する
2. 各ファイルのフロントマター（status, priority）を読み取り、一覧に表示する
3. `status: idea` のものを優先的に上に表示する
4. ユーザーに番号で選んでもらう（複数選択も可）

### Step 3: 投稿形式の判断

ネタの内容を分析し、最適な投稿形式を提案する:

| 形式 | 向いているネタ | サイズ |
|---|---|---|
| 単発投稿 | 単発の気づき・共感・問いかけ・断言 | 100~140文字 |
| スレッド | ステップ解説・Before/After・比較・ストーリー | 2~3ツイート x 100~140文字 |

ユーザーに形式を確認し、希望があれば変更する。

### Step 3.5: 自分の経験ヒアリング（海外事例・外部ソースの場合）

ネタファイルの `source` が `interview` 以外（海外事例・ニュース・Tips等）の場合、下書き生成前にユーザーに聞く:

**「このネタに関連して、自分やクライアントで似た経験ある？ あれば投稿に織り込みたい。」**

- 回答があれば → その経験を投稿の軸にし、海外事例は補強材料として使う
- 「特にない」→ 「自分もこれから試す」「うちでも検討中」等、発信者の立場を最低1文入れる形で生成する
- いずれにせよ、**「誰が書いても同じ」投稿にならないようにする**

### Step 4: 下書き生成（複数パターン）

選択されたネタファイルを読み込み、**2パターンの下書き**を生成する。
各パターンは異なるフックの型と構成パターンを使い、切り口を変える。

**上記「投稿ルール」セクションの全ルールに従って下書きを生成する。**

#### トーンのキャリブレーション

投稿案から抽出した「AI将軍のトーン」:

```
○ AI将軍のトーン:
「AI導入の見積もり出してくる会社、9割ボッタクリです。」
「部下に仕事教えるのと全く同じ。最初だけ手間かかって、あとはどんどん楽になる構造です。」
「プログラミングの知識ゼロでいい。」
「怖いのは「AIを使ってる会社」じゃなくて、「AIを使ってることに気づかないまま負ける」ことです。」

x AI将軍じゃないトーン:
「AIの活用により業務効率化が期待できます」（記事調すぎ）
「マジでヤバいww 最強すぎるww」（カジュアルすぎ）
「知らない人は全員バカです」（攻撃的）
```

### Step 5: パターン提示

#### 単発投稿の場合

```
## パターンA: {構成パターン名}（{フックの型}）

**投稿（{文字数}文字）:**

{本文}

**引き込み分析:** {使用した武器}

---

## パターンB: {構成パターン名}（{フックの型}）
...
```

#### スレッドの場合

```
## パターンA: {構成パターン名}（{フックの型}）

**1/{N}:**
{本文}

**2/{N}:**
{本文}

...

**{N}/{N} CTA:**
{本文}

**引き込み分析:** {使用した武器}
```

提示後、ユーザーに聞く:
**「どのパターンがいい？ そのまま使う or 修正したい部分ある？」**

### Step 6: フィードバック対応

ユーザーからのフィードバックに応じて修正する。
納得いくまで何度でも修正する。

### Step 7: 投稿確定

ユーザーが「OK」「これでいこう」「投稿する」等と言ったら:

**「この内容で投稿済みにしていい？」** と最終確認する。

確認が取れたら以下を実行:

1. **投稿案ファイルに保存**
   - 保存先: `SNS/{アカウント名}/投稿案_{YYYYMMDD}.md`
   - 同じ日付のファイルが既にある場合は末尾に追記する

2. **元の投稿ネタファイルのステータスを更新**
   - `status: idea` → `status: posted`
   - `posted_date:` → `posted_date: {YYYY-MM-DD}`
   - `platform: X` を追加

3. **投稿済みフォルダに移動**
   - 移動先: `SNS/{アカウント名}/投稿済み/`（フォルダがなければ作成）
   - 元の `投稿ネタ/` から `投稿済み/` にファイルを移動する

4. **投稿時のリマインド**
   - **「投稿後15~30分以内に来たリプライに必ず返信してください（リーチ2倍）」**
   - 推奨投稿時間帯: 朝6~8時 / 昼12~13時 / 夜20~22時（木曜夜がベスト）
   - 残りの未投稿ネタ（`status: idea`）の件数

### 投稿案ファイルのフォーマット

```markdown
---

## 投稿{N}【X】: {構成パターン名}（元ネタ: {ネタファイル名}）

**形式:** {単発 / スレッド}
**引き込み:** {使用した武器}

**投稿:**
{本文}

（スレッドの場合）
**2ツイート目:**
{本文}
...
```

### 投稿前チェック（下書き生成時に自動で確認）

#### 密度弾チェック（最優先）
- [ ] 冒頭10~20文字でスクロールが止まるか？（0.3秒テスト）
- [ ] 不完全フック・断言カット・ギャップブリッジのいずれかが効いているか？
- [ ] 1行目で結論を全部言い切っていないか？
- [ ] 余白があるか？（全部語りきっていないか）
- [ ] 140文字以内に収まっているか？（スレッドの場合は各ツイート）

#### 構成チェック
- [ ] PAS / BAB / ストーリー型のいずれかの構成か？
- [ ] 意味ブロック+面積支配の改行になっているか？（1行20文字前後、適切な空行配置）
- [ ] 体言止め・短文で断言しているか？（弱い主張になっていないか）
- [ ] スレッドの場合、各ツイートが独立して価値があるか？

#### コンテンツチェック
- [ ] 最終投稿に問いかけ or CTA があるか？（リプ誘導最優先）
- [ ] 経営者の関心（売上・コスト・人材・時間）に着地しているか？
- [ ] 専門用語を使っていないか？
- [ ] 記事っぽく・宣伝っぽくなっていないか？
- [ ] 「過去の自分＝読者の今」として語れているか？
- [ ] 投稿案と同じトーンになっているか？
- [ ] 外部リンクが本文にないか？
- [ ] ハッシュタグがついていないか？
- [ ] **「自分/うちでは」の一人称要素が入っているか？**（海外事例の翻訳だけで終わっていないか。発信者の存在感がある投稿になっているか）

#### 炎上チェック
- [ ] 炎上防止3原則に違反していないか？
- [ ] 特定の個人・企業・サービスを否定していないか？
- [ ] 読者を見下す表現になっていないか？
- [ ] 攻撃的なトーンになっていないか？（Grok検知対策）
- [ ] この投稿を見た関係者が不快に感じないか？
- [ ] 命令口調（「しろ」「するな」等）を使っていないか？

#### アルゴリズムチェック
- [ ] 外部リンクが本文に含まれていないか？（リンクがある場合はセルフリプに）
- [ ] タグはつけていないか？
- [ ] 投稿後15~30分以内にリプ返しできる時間帯か？
- [ ] リンクや補足がある場合、セルフリプの内容は準備できているか？
- [ ] ナビゲーション要素は適切な場面でのみ使っているか？（毎回使わない）
- [ ] 引用RTされやすい余地があるか？（70/30ルール）
- [ ] 体験会・セミナー・イベントへの誘導は、ユーザーから明示的に指示がない限り書いていないか？

---

## 使用例

```
/x-draft
→ アカウント選択 → ネタ一覧 → 形式提案 → 複数パターン生成 → 選択 → 投稿済み

/x-draft SNS/AI将軍/投稿ネタ/260525-ChatGPTとClaudeCodeの違い.md
→ 指定ファイルから直接下書き生成

/x-draft AI将軍
→ アカウント指定 → ネタ一覧 → 選択 → 下書き生成
```
